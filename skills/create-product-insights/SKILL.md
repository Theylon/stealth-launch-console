# Create Product Insights

## Overview

Turn fragmented customer and market evidence into prioritized product insight. Use this to understand PMF gaps, surface voice-of-customer patterns, find messaging opportunities, and recommend roadmap moves.

Read [references/framework.md](references/framework.md) for the section order and scoring model.

## Workflow

1. Inventory the available sources and note which ones are weak or anecdotal.
2. Assess product-market-fit indicators and major gaps.
3. Synthesize praise, pain, request, and usage patterns.
4. Compare the product against competitors and identify opportunity gaps.
5. End with prioritized roadmap and messaging recommendations.

## Output Contract

- Start directly with the framework output.
- Use tables when the reference calls for them.
- Separate observed customer evidence from inferred recommendations.
- Apply explicit prioritization logic to feature and roadmap recommendations.

## Quality Bar

- Be data-driven when data exists.
- If metrics are missing, say what evidence is missing instead of faking precision.
- Prioritize only the changes that clearly connect to retention, growth, or value realization.
