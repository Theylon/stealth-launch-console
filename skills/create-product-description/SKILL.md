# Create Product Description

## Overview

Write product descriptions that scale from short catalog blurbs to full product pages, while keeping benefits, proof, and SEO aligned. Use this when the user needs multi-format product copy rather than just one paragraph.

Read [references/framework.md](references/framework.md) for the length variants and channel adaptations.

## Workflow

1. Confirm the product, target buyer, features, and price position.
2. Define the core positioning and differentiator.
3. Create the short, medium, full, and extended versions requested by the framework.
4. Add benefit-led blocks, use cases, proof, SEO, and objections.
5. Adapt the copy for ecommerce, social commerce, and email use.

## Output Contract

- Start directly with the product description framework.
- Lead with benefits before features.
- Keep the format scannable with bullets and short blocks.
- Integrate keywords naturally.

## Quality Bar

- Each description length should feel purpose-built for its channel.
- Technical detail should support purchase confidence, not overwhelm the copy.
- Use cases and proof should feel specific to the product and buyer.
