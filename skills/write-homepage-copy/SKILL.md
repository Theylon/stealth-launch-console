# Write Homepage Copy

## Overview

Write a homepage that expresses the approved positioning and message architecture without slipping into generic SaaS copy. This skill should create section-level copy and narrative flow, not just a hero line.

Read [references/frameworks.md](references/frameworks.md) when you need awareness-stage and direct-response guidance.

Use external page benchmarks selectively:

- Use [Refero](https://refero.design/) to sanity-check homepage section order, proof placement, CTA rhythm, and common page archetypes.
- Use [component.gallery](https://component.gallery/) only if the user asks for guidance that depends on specific modules such as nav, pricing, FAQ, tabs, cards, or forms.
- Keep the skill copy-first. These sources should shape structure only when they help the narrative land more clearly.

## Workflow

1. Confirm the target buyer, awareness stage, and homepage goal.
2. If needed, use Refero to benchmark the likely homepage pattern for the category and buying motion.
3. Pull the strongest promise, proof, and objections from the message architecture.
4. Draft the page in section order: hero, problem, solution, proof, how it works, objections, CTA.
5. Match section depth to the buying motion. A sales-led homepage can ask for a stronger CTA than an early-stage product with weaker proof.
6. Tighten for clarity and remove generic filler.

## Output Contract

Always produce these sections:

1. `Homepage Strategy`
   - Buyer, awareness stage, page goal, CTA goal
2. `Section Outline`
   - Recommended section order with purpose
3. `Homepage Draft`
   - Hero
   - Subhead
   - Section copy
   - CTA copy
4. `Rationale`
5. `Missing Assumptions`
6. `Suggested Next Skill`
   - Usually `$audit-messaging` or `$write-email-sequence`

## Quality Bar

- The hero must say what the product helps the customer do.
- Each section must earn its place on the page.
- Use proof before asking for trust.
- Avoid empty phrases such as "streamline workflows," "unlock efficiency," or "transform your business" unless grounded in specifics.
- Do not let benchmark examples flatten the positioning into generic category copy.

## Skill Chain

Default handoff: `$audit-messaging`
