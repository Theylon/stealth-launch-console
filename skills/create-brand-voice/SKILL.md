# Create Brand Voice

## Overview

Define how the brand should sound across channels and audiences. Use this when the user needs concrete voice standards, tone ranges, vocabulary rules, and before-and-after examples that content teams can follow consistently.

Read [references/framework.md](references/framework.md) for the voice-guide structure.

## Workflow

1. Confirm the company, product, audience, and any existing brand values.
2. Define the core personality traits and tone dimensions.
3. Translate those traits into practical rules, vocabulary choices, and channel adaptations.
4. Provide transformation examples that show the difference between generic and on-brand language.
5. End with a review checklist that writers can actually use.

## Output Contract

- Start directly with the voice guide.
- Include ratings, examples, and concrete do-not-do rules.
- Make the guidance specific enough to edit copy against.
- Prefer operational clarity over abstract brand language.

## Quality Bar

- Every trait should change writing behavior.
- Do and do-not pairs should be specific.
- Channel shifts should preserve the same core voice.
