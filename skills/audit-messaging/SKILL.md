# Audit Messaging

## Overview

Diagnose messaging quality against the strategy stack that produced it. This skill should identify why the copy fails, rank the issues, and suggest fixes that connect back to ICP, positioning, value, and proof.

Read [references/frameworks.md](references/frameworks.md) when you need the evaluation lenses.

## Workflow

1. Confirm the target asset, audience, and intended outcome.
2. If available, compare the copy against the market context, ICP choice, positioning, and message architecture.
3. Evaluate clarity, differentiation, relevance, proof, objection handling, and awareness-stage fit.
4. Rank the findings by severity and by how much they block conversion or comprehension.
5. Recommend fixes at the right level: strategy, messaging architecture, or line edit.

## Output Contract

Always produce these sections:

1. `Overall Verdict`
   - Short summary of what is working and what is not
2. `Findings`
   - Severity-ranked issues with evidence from the copy
3. `Recommended Fixes`
   - Tie each fix to the source issue
4. `Rationale`
5. `Missing Assumptions`
6. `Suggested Next Skill`
   - Usually the skill that should repair the deepest issue

## Quality Bar

- Quote or point to the exact lines that create the problem.
- Distinguish strategy failures from wording failures.
- Do not ask for a line edit when the real issue is weak positioning or no proof.
- Prioritize the few fixes most likely to improve comprehension and conversion.

## Skill Chain

This is usually the last step in the v1 chain, but it may route back upstream when the issue is strategic.
