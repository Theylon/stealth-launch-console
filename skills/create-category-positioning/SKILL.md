# Create Category Positioning

## Overview

Create a category strategy when the company wants to name, reframe, or dominate a market category instead of competing only at the product level. Use this for category POV, naming, lightning-strike rollout, and competitive response planning.

Read [references/framework.md](references/framework.md) for the category design structure.

## Workflow

1. Assess the current category landscape and buyer expectations.
2. Evaluate whether the best move is creation, redesign, carve-out, or category combination.
3. Design the new category name, definition, and point of view.
4. Map differentiation, rollout phases, ecosystem development, and messaging.
5. Finish with the competitive response playbook.

## Output Contract

- Start directly with the framework output.
- Make the positioning bold but plausible.
- Keep timelines and category-adoption tactics realistic.
- Prefer one recommended category move with clear rationale.

## Quality Bar

- Avoid invented categories that add no buying clarity.
- Tie the category story to a real problem and a real buyer shift.
- Give the company a credible path to own the space.
