# Create Pricing Strategy

## Overview

Create a full pricing strategy that connects product value, segmentation, competitive position, tier design, and packaging. Use this when the user needs more than a pricing page rewrite and needs the underlying pricing logic as well.

Read [references/framework.md](references/framework.md) for the required sections and decision criteria.

## Workflow

1. Confirm the product, segments, current pricing, value metric, and competitor context.
2. Evaluate candidate value metrics and recommend one.
3. Design the competitive position and tier architecture.
4. Add pricing psychology, trial or freemium logic, discount governance, and expansion pathways.
5. Finish with a price-testing roadmap and page recommendations.

## Output Contract

- Start directly with the pricing framework.
- Show the rationale for the chosen value metric and tier structure.
- Keep assumptions visible where hard pricing data is missing.
- Use tables where the framework calls for them.

## Quality Bar

- Pricing should scale with customer value.
- Tiering should create clear upgrade paths.
- Avoid decorative pricing tactics that do not match the product or market.
