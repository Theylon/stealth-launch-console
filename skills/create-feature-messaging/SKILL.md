# Create Feature Messaging

## Overview

Translate one feature into outcome-led messaging across launch surfaces. Use this when a single feature needs its own positioning, benefits, objections, proof, and channel-specific copy rather than being buried inside general product messaging.

Read [references/framework.md](references/framework.md) for the required ladders, copy blocks, and launch sequence.

## Workflow

1. Confirm the feature, target audience, and relevant alternatives.
2. Build the feature positioning statement and benefit ladder.
3. Draft headline options, short descriptions, and use-case messaging.
4. Add objections, comparisons, proof, and channel-specific adaptations.
5. Finish with the launch sequence.

## Output Contract

- Start directly with the framework output.
- Keep benefits ahead of technical detail.
- Make the messaging persona-specific.
- Support claims with proof points or proof requirements.

## Quality Bar

- One feature, one coherent story.
- Use tangible outcomes wherever possible.
- Keep channel assets consistent with the same core promise.
