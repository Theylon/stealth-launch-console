# Create Buyers Journey

## Overview

Map the buyer journey from unawareness to decision, including stage mindset, content needs, objections, triggers, and measurement. Use this when the user needs a stage-based content and conversion plan rather than a generic funnel.

Read [references/framework.md](references/framework.md) for the five-stage journey structure and tables.

## Workflow

1. Confirm the audience, product, and any sales-cycle context.
2. Map the five journey stages and the emotional or informational shifts between them.
3. Add touchpoints, content needs, objections, conversion triggers, and friction points.
4. Tie each stage to measurable outcomes.
5. Keep the journey specific to the company and audience instead of generic lifecycle advice.

## Output Contract

- Start directly with the journey framework.
- Use the five stages from the reference.
- Include 15 to 20 touchpoints and 20 or more stage-mapped content ideas when possible.
- Keep stage transitions explicit.

## Quality Bar

- The journey should sound like this buyer, not any buyer.
- Objections and triggers should match the stated audience and motion.
- Measurement should be operational, not vanity-only.
