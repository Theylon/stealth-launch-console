# Create FAQs

## Overview

Build a complete FAQ system that handles customer questions, sales objections, and organic-search opportunities. Use this when the user needs more than a short Q&A block and wants a maintainable FAQ program.

Read [references/framework.md](references/framework.md) for the category structure and schema requirements.

## Workflow

1. Confirm the company, offer, audience, and any known recurring questions.
2. Group the FAQ set by question category.
3. Write direct answers first, then expand with supporting detail and next steps.
4. Convert core objections into FAQ-friendly questions.
5. Add SEO, schema, maintenance, and channel-adaptation layers.

## Output Contract

- Start directly with the FAQ framework.
- Answer the question directly before adding context.
- Use natural-language phrasing that mirrors how users ask.
- Include schema-ready formatting when requested by the framework.

## Quality Bar

- Avoid defensive language.
- Each answer should be clear enough to reduce support or sales friction.
- Keep related-question links and next steps useful, not filler.
