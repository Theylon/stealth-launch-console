# Create ICP Sizing

## Overview

Define who the company should target and how large that opportunity is. Use this when the user needs a quantified ICP, tiering model, qualification criteria, and realistic market-sizing logic in one output.

Read [references/framework.md](references/framework.md) for the full section list and sizing requirements.

## Workflow

1. Normalize the company, offer, geography, and customer examples.
2. Define the firmographic, technographic, behavioral, and organizational fit criteria.
3. Map problem-solution fit and buying signals.
4. Estimate TAM, SAM, and SOM using a visible methodology and cited inputs.
5. Build the tiering and scoring model that sales or marketing can use operationally.

## Output Contract

- Start directly with the framework output.
- Quantify criteria wherever the context allows.
- Show how sizing was calculated.
- Cite sources for market-size data when external data is used.

## Quality Bar

- Avoid vague ICP language.
- Keep the scoring model operational, not decorative.
- Do not fake precision when sizing data is weak.
