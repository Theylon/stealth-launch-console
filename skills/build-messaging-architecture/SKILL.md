# Build Messaging Architecture

## Overview

Turn strategic inputs into a message house that copywriting skills can reuse without rethinking the fundamentals. This skill is the bridge between positioning and page-level writing.

Read [references/frameworks.md](references/frameworks.md) when you need the StoryBrand and Made to Stick filters.

## Workflow

1. Confirm the positioning core and recommended value proposition.
2. Write one primary promise that captures the customer outcome.
3. Build three supporting message pillars, each backed by proof.
4. Add objection handling, audience-specific proof cues, and CTA language.
5. Remove claims that are interesting internally but weak externally.

## Output Contract

Always produce these sections:

1. `Core Message`
   - One-liner
   - Primary promise
2. `Message House`
   - Three support pillars with proof
3. `Objection Handling`
   - Top objections and the response angle
4. `CTA Language`
   - Primary and secondary CTA options
5. `Rationale`
6. `Missing Assumptions`
7. `Suggested Next Skill`
   - Usually `$write-homepage-copy`, `$write-landing-page-copy`, or `$write-email-sequence`

## Quality Bar

- Keep the core message understandable on first read.
- Each supporting pillar must answer "why believe this?"
- Use proof with the closest possible connection to the claim.
- Do not let the message house drift away from the approved positioning.

## Skill Chain

Default handoff: one of the copy skills
