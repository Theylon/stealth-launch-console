# Create Landing Page

## Overview

Create a comprehensive landing page that uses persuasion frameworks, proof, objections, and CTA sequencing to drive conversion. Use this when the user wants a long-form conversion page rather than the lighter `$write-landing-page-copy` skill.

Read [references/framework.md](references/framework.md) for the required sections and persuasive structure.

When needed, use external benchmarks to keep the page structure realistic:

- Use [Refero](https://refero.design/) to sanity-check landing-page section patterns, offer presentation, proof placement, and CTA pacing.
- Use [component.gallery](https://component.gallery/) only if you are recommending specific modules such as pricing tables, FAQ accordions, tabs, forms, or comparison blocks.
- Keep these sources subordinate to the positioning and offer strategy. They inform structure, not the core message.

## Workflow

1. Confirm the offer, audience, value proposition, and any competitive context.
2. If page structure is not already obvious, review relevant Refero examples to benchmark section order and content density.
3. Build the page from hero through final CTA in the order defined by the framework.
4. Layer in psychological structure, proof, objections, urgency, authority, and risk reversal.
5. Keep the copy specific and persona-driven throughout.
6. End with footer trust elements rather than commentary.

## Output Contract

- Output structured markdown only.
- Start directly with the hero section.
- End with the footer trust elements.
- Include specific copy for each section, not placeholders.

## Quality Bar

- Every section should move the reader toward action.
- Use ethical urgency and believable proof.
- Keep the page scannable for mobile consumption.
- Do not let benchmark examples pull the page away from the approved offer or audience.
