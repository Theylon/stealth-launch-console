# Create Referral Program

## Overview

Design a referral program that balances customer motivation, program economics, and operational mechanics. Use this when the user needs a full program design rather than a few reward ideas.

Read [references/framework.md](references/framework.md) for the required metrics, viral-loop structure, and rollout plan.

## Workflow

1. Confirm the business model, LTV, CAC, referral goals, and primary user motivation.
2. Model the incentive budget and choose the program structure.
3. Design the viral loop, sharing channels, and user journeys.
4. Add attribution, fraud prevention, messaging, and implementation phases.
5. Keep rewards and targets grounded in the business economics.

## Output Contract

- Start directly with the referral framework.
- Include a concrete recommendation for referrer and referee incentives.
- Keep the K-factor logic visible.
- Show how the program will be tracked and governed.

## Quality Bar

- Incentives should be economically rational.
- Program mechanics should be simple enough to launch.
- Reward timing and qualification rules must be explicit.
