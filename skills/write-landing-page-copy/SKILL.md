# Write Landing Page Copy

## Overview

Write a single-purpose landing page that converts one audience on one offer. This skill should stay narrower and more conversion-focused than the homepage skill.

Read [references/frameworks.md](references/frameworks.md) when you need the awareness and offer-specific copy rules.

Use external page benchmarks selectively:

- Use [Refero](https://refero.design/) to sanity-check landing-page composition, offer framing, proof sequencing, and CTA pacing.
- Use [component.gallery](https://component.gallery/) only if you need to recommend or describe specific modules such as pricing tables, FAQ accordions, comparison blocks, forms, or tabs.
- Keep the skill copy-first. Structural inspiration is allowed; copy should still come from the offer, audience, and awareness stage.

## Workflow

1. Confirm the offer, target audience, traffic source, and CTA.
2. If needed, benchmark relevant landing-page patterns in Refero before locking the section order.
3. Match the page to one awareness stage and one core promise.
4. Build the page around the specific offer: feature, use case, webinar, demo, migration, or campaign.
5. Sequence the copy to handle motivation, friction, proof, and action.
6. Remove any section that weakens focus or introduces a second agenda.

## Output Contract

Always produce these sections:

1. `Landing Page Strategy`
   - Audience
   - Offer
   - Awareness stage
   - Conversion goal
2. `Section Outline`
3. `Landing Page Draft`
4. `Rationale`
5. `Missing Assumptions`
6. `Suggested Next Skill`
   - Usually `$audit-messaging` or `$write-email-sequence`

## Quality Bar

- Keep one page, one audience, one promise, one CTA path.
- Make the offer concrete and the next step obvious.
- Match the amount of explanation to the buyer's awareness stage.
- Use proof close to the ask.
- Do not let benchmark patterns introduce a second audience, a second offer, or generic filler sections.

## Skill Chain

Default handoff: `$audit-messaging`
