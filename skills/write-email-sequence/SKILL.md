# Write Email Sequence

## Overview

Write short, coherent email sequences that match the buyer's awareness stage and the business goal. Default to concise B2B SaaS emails unless the user explicitly asks for longer narrative copy.

Read [references/frameworks.md](references/frameworks.md) when you need the sequence and awareness rules.

## Workflow

1. Confirm the sequence type, audience, stage, and CTA.
2. If the user does not specify length, default to four emails.
3. Give each email one job: open the loop, frame the problem, provide proof, handle friction, or ask for action.
4. Keep the sequence cumulative so each email advances the argument instead of repeating it.
5. Tune subject lines, body length, and CTA strength to the audience and motion.

## Output Contract

Always produce these sections:

1. `Sequence Strategy`
   - Sequence type
   - Audience
   - Awareness stage
   - CTA goal
2. `Email Map`
   - One-line purpose for each email
3. `Email Drafts`
   - Subject line
   - Preview text if useful
   - Body copy
   - CTA
4. `Rationale`
5. `Missing Assumptions`
6. `Suggested Next Skill`
   - Usually `$audit-messaging`

## Quality Bar

- One email, one job.
- Keep the emails skimmable and specific.
- Use proof and specificity before escalation.
- Do not let the sequence drift into generic nurture language.

## Skill Chain

Default handoff: `$audit-messaging`
