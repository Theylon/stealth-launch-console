# Create Messaging

## Overview

Build a reusable messaging system for the full website, not just a page draft. Use this when the user needs modular copy, audience-specific variations, page hierarchy, trust placement, and a scalable messaging foundation across marketing surfaces.

Read [references/framework.md](references/framework.md) for the required sitewide and page-level sections.

Use external page benchmarks carefully:

- Use [Refero](https://refero.design/) to pressure-test page hierarchy, module expectations, and the realism of proposed website sections.
- Use [component.gallery](https://component.gallery/) only when the messaging system needs to account for a specific component type such as pricing, FAQs, tabs, forms, cards, or navigation patterns.
- Keep this skill strategy-first. The benchmark should inform packaging and placement of messages, not replace the positioning work.

## Workflow

1. Confirm the company, product, audience, and any approved positioning.
2. Establish sitewide foundations first: positioning, key messages, segment matrix, keywords, and trust signals.
3. If the site structure is still fuzzy, review relevant Refero examples to benchmark page types and module expectations.
4. Build the main website sections in descending importance.
5. Keep the copy modular so it can be reused across pages, ads, sales, and enablement.
6. Maintain clarity, scannability, and trust near the CTA.

## Output Contract

- Output markdown only.
- Start directly with the deliverables.
- Include both strategic foundations and page-level copy frameworks.
- Prefer modular copy blocks over polished prose walls.

## Quality Bar

- Clarity first.
- Benefits before features.
- Make the structure easy to scan and adapt.
- Use social proof close to decision points.
- Do not let inspiration research override the core audience segmentation or message hierarchy.
