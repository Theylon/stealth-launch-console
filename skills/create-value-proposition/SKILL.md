# Create Value Proposition

## Overview

Build 5 to 7 value ladders that translate what the product has into what the customer achieves. Use this when feature lists need to become value messaging, role-based positioning, or website hierarchy.

Read [references/framework.md](references/framework.md) for the ladder structure and downstream sections.

## Workflow

1. Normalize the company, product, and target segments.
2. Identify the most important product features or capabilities.
3. Convert each feature into a capability, benefit, outcome, and proof point.
4. Adapt the ladders for executive, manager, and user audiences.
5. End with website messaging guidance and proof strategy.

## Output Contract

- Start directly with the framework output.
- Include 5 to 7 complete value ladders.
- Use customer language rather than internal jargon.
- Support each ladder with a proof point or a proof requirement if evidence is missing.

## Quality Bar

- Connect features to business outcomes, not just convenience.
- Make role-based differences explicit.
- Prioritize the ladders that matter most to the target audience.
