# Value Proposition Frameworks

Use these lenses to connect product strengths to buyer progress.

## Value Proposition Design

- Start with the customer profile: jobs, pains, gains.
- Map only the product features that actually relieve those pains or create those gains.
- A good value proposition is relevant, differentiated, and credible.

## Competing Against Luck

- Frame the problem as a job the customer is trying to get done.
- Look for the progress the customer wants, not just the functionality they mention.
- Buying friction matters; emotional and social forces can be as important as functional ones.

## Practical rules

- Keep the recommended value proposition to one core promise.
- Treat proof as part of the value proposition, not as a later copywriting step.
- If multiple jobs appear, choose the one most tied to purchase intent.
