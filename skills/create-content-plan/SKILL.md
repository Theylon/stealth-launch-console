# Create Content Plan

## Overview

Build a content plan that ties audience needs, themes, distribution, and measurement into one operating document. Use this when the user needs an actionable content system rather than a loose list of topics.

Read [references/framework.md](references/framework.md) for the required deliverables and calendar structure.

## Workflow

1. Confirm the company, audience, and any keyword or topic inputs.
2. Write the reader story and voice signals that anchor the plan.
3. Build the pillar matrix, source list, and content-type distribution.
4. Define measurable metrics.
5. Produce the 20-day strategic calendar with cross-promotion logic.

## Output Contract

- Start directly with the framework output.
- Keep the recommendations specific and measurable.
- Make the calendar concrete enough to execute.
- Use percentage splits only when they match the proposed strategy.

## Quality Bar

- Content pillars should map to audience needs and business goals.
- Distribution should reflect resource reality.
- Metrics should separate awareness, engagement, conversion, and retention.
