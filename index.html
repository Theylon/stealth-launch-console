Repo overview

A skill library with an operating spine.

The root documents define the service, its execution protocol, and its commercial packaging. The `skills/` directory is the real product inventory, and the dogfood client workspace proves the artifact model with approved outputs already saved on disk.

Why this MVP fits

It uses the strongest existing chain.

  • `$capture-market-context` normalizes the launch context.
  • `$choose-ideal-customer` sharpens the wedge for the announcement.
  • `$position-product` defines the alternative and differentiated frame.
  • `$define-value-proposition` turns that into a credible promise.
  • `$create-press-release` packages the result into a real launch document.
What comes next

Backend orchestration, not a different front end.

  • Swap deterministic templates for actual skill execution.
  • Save each approved run into `clients/<client-slug>/` automatically.
  • Add collaboration, refinement loops, and publish workflows.
Hosting model

Keep the first external version static and easy to ship.

Because the app is a single `index.html` with no build step, GitHub Pages is the right first hosting target. Push the repo, enable Pages via the included workflow, and you have a shareable MVP without introducing infrastructure debt before the product surface is validated.
